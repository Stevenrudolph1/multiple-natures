# Visual & Design Directive — Multiple Natures Website

**Version 1.0 | January 2026**

---

## Core Principle

Multiple Natures is an observational framework, not an identity system, not a transformation program, and not an assessment product.

The visual design must reduce emotional signaling, avoid implication, and support careful reading.

**If a visual element suggests discovery, identity, progress, or transformation, it is wrong.**

---

## Absolute Rules (Non-Negotiable)

### 1. No Photography by Default

- Do not use stock photos
- Do not use inspirational imagery
- Do not use people, faces, paths, horizons, gestures, or "moment" imagery
- Do not use photos to set mood or tone

**The site should function fully without any images.**

### 2. No Iconography for Concepts

- No icons for Natures
- No icons for Intelligences
- No symbols representing traits, types, or categories
- No visual shorthand that implies classification

**Icons imply stable identity. MN rejects this.**

### 3. No Color Coding by Concept

- No colors assigned to Natures
- No colors assigned to Intelligences
- No visual legend
- No dashboard-style segmentation

**Color must never answer a conceptual question.**

### 4. No Card-Based or Tile-Based Layouts

Avoid layouts that imply:

- Selection
- Comparison
- Modular identity
- "Choose your type"

**Use linear, readable, document-style layouts.**

---

## Required Design Posture

### Text-First

- Typography does the work
- Hierarchy through spacing and type, not graphics
- Pages should feel like a field guide or reference manual

### Slow Reading

- Generous margins
- Clear vertical rhythm
- Calm, neutral presentation
- Minimal emphasis

**If the page feels skimmable, it is too optimized.**

---

## Typography Guidelines

- One primary readable typeface (serif or humanist sans)
- One secondary neutral sans for navigation
- No display fonts
- No decorative type
- No exaggerated weights or styles

**Typography should encourage attention, not personality.**

---

## Color Guidelines

- Neutral base palette
- One muted accent color, used sparingly
- Color is environmental, not semantic
- Color should never imply meaning

**If removing color changes interpretation, the design is wrong.**

---

## Acceptable Visual Elements (Optional)

Only the following are allowed:

- Whitespace as structure
- Simple typographic separators
- Subtle, non-symbolic geometric marks (lines, dots)
- Repetition of neutral layout motifs

**These must never be explained or referenced in content.**

---

## What This Site Must Feel Like

**YES:**
- A reference
- A framework
- A language guide
- A boundary-setting document

**NOT:**
- A personality assessment
- A coaching brand
- A transformation promise
- A motivational experience

---

## Tests to Apply Before Finalizing Any Page

Ask:

1. Does this visual imply identity?
2. Does this visual imply progress or improvement?
3. Does this visual suggest "before / after"?
4. Does this visual make the concept feel owned, possessed, or claimed?

**If yes to any → remove it.**

---

## Final Check

If someone can:

- Skim the page
- Feel inspired
- Think "this explains me"

**Then the design has failed.**

The site should reward careful readers, not quick consumers.

---

## Implementation Notes

### For HTML/CSS Development

- Use semantic HTML5 structure
- CSS Grid or Flexbox for layout, but favor single-column reading flow
- Maximum content width: 680-720px for body text
- Generous line-height (1.5-1.7)
- Paragraph spacing that encourages pause

### Logo Specifications

**IMPORTANT: Do not change logo size without explicit approval.**

- **Header logo height: 120px** (this is the correct size - do not reduce)
- Logo must not overflow its container
- If header needs adjustment, adjust header padding, not logo size
- Logo file: `images/mn-logo.png` (copied from MN-primary-horizontal-logo.png)

### For Content Generation

- All page content must comply with Canon Docs 01-07
- Visual presentation must comply with this directive (09)
- No element should do conceptual work that text should do
