# MN Project Governance — Summary

**Created:** 2026-02-03

## What Was Created

Canonical governance documents for all Multiple Natures work:

1. **MN-GOV-00** - Core MN theory and philosophy
2. **MN-GOV-01** - Constraints on all MN work  
3. **MN-GOV-10** - MNTEST technical specifications
4. **MN-GOV-11** - MNTEST usage governance
5. **MN-GOV-20** - Licensing framework

## Key Principles

1. **Person Over System** - No MN system may outrank the person
2. **Observation, Not Authority** - MN notices patterns, doesn't decide
3. **Context Determines Value** - Assets/liabilities, never strengths/weaknesses
4. **MN Stops at Orientation** - Continuing past orientation adds pressure
5. **Coherence Over Revenue** - Governance doesn't change for commercial appeal
6. **Evidence Over Preference** - Changes based on use, not argument
7. **Ideas Open, Implementations Protected** - Theory public, tools licensed

## What This Enables

- Clear boundaries for appropriate/inappropriate use
- Quality standards for MN work
- Licensing clarity for derivative tools
- Protection from distortion
